# <a name="creating-an-object-through-a-class-object"></a>透過類別物件建立物件

隨著電腦網路的重要性日益增加，用戶端和伺服器都必須能夠輕鬆且有效率地進行互動，不論它們是位於同一部電腦或網路上。 重要的是，用戶端可以啟動伺服器、建立伺服器物件的實例，並且可以存取物件上的介面方法。

COM 提供此基本 COM 進程的延伸模組，讓它在網路上幾乎順暢。 如果用戶端能夠透過 CLSID 來識別伺服器，則呼叫一些簡單的函式會允許 COM 執行所有尋找和啟動伺服器的工作，以及啟始物件。 登錄中的子機碼可讓遠端伺服器註冊其位置，因此用戶端不需要該資訊。 對於想要利用網路功能的應用程式，COM 物件建立功能可提供更大的彈性和效率。

如需詳細資訊，請參閱下列主題：

-   [COM 類別物件和 Clsid](com-class-objects-and-clsids.md)
-   [尋找遠端物件](locating-a-remote-object.md)
-   [實例建立 Helper 函數](instance-creation-helper-functions.md)
