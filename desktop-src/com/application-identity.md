# <a name="application-identity"></a>應用程式識別碼

應用程式的身分識別是用來執行應用程式的帳戶。 身分識別可以是目前登入 (互動式使用者) 的使用者、啟動伺服器的使用者、指定的使用者或服務。 若要瞭解如何使用 Dcomcnfg.exe 設定應用程式識別，請參閱 [使用 DCOMCNFG 設定 Processwide 安全性](setting-processwide-security-using-dcomcnfg.md)。

下列主題描述四種類型的應用程式識別，並說明每種類型的限制：

-   [互動式使用者](interactive-user.md)
-   [正在啟動使用者](launching-user.md)
-   [指定的使用者](specified-user.md)
-   [服務身分識別](service-identity.md)
