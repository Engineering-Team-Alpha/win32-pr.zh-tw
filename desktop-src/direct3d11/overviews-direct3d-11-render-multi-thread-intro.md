# <a name="introduction-to-multithreading-in-direct3d-11"></a><span data-ttu-id="01ca7-103">Direct3D 11 中的多執行緒簡介</span><span class="sxs-lookup"><span data-stu-id="01ca7-103">Introduction to Multithreading in Direct3D 11</span></span>

<span data-ttu-id="01ca7-104">多執行緒的設計目的是要使用一或多個執行緒同時執行工作，以改善效能。</span><span class="sxs-lookup"><span data-stu-id="01ca7-104">Multithreading is designed to improve performance by performing work using one or more threads at the same time.</span></span>

<span data-ttu-id="01ca7-105">在過去，這通常是藉由產生單一主執行緒來進行轉譯，以及一或多個執行緒來執行準備工作（例如物件建立、載入、處理等）。</span><span class="sxs-lookup"><span data-stu-id="01ca7-105">In the past, this has often been done by generating a single main thread for rendering and one or more threads for doing preparation work such as object creation, loading, processing, and so on.</span></span> <span data-ttu-id="01ca7-106">不過，使用 Direct3D 11 內建的同步處理功能時，多執行緒處理背後的目標是要利用每個 CPU 和 GPU 迴圈，而不需讓處理器等待另一個處理器 (特別是因為它會直接影響畫面播放速率) 。</span><span class="sxs-lookup"><span data-stu-id="01ca7-106">However, with the built in synchronization in Direct3D 11, the goal behind multithreading is to utilize every CPU and GPU cycle without making a processor wait for another processor (particularly not making the GPU wait because it directly impacts frame rate).</span></span> <span data-ttu-id="01ca7-107">如此一來，您就可以產生最大量的工作，同時維持最佳的畫面播放速率。</span><span class="sxs-lookup"><span data-stu-id="01ca7-107">By doing so, you can generate the most amount of work while maintaining the best frame rate.</span></span> <span data-ttu-id="01ca7-108">因為 API 會執行同步處理，所以不再需要呈現單一框架的概念。</span><span class="sxs-lookup"><span data-stu-id="01ca7-108">The concept of a single frame for rendering is no longer as necessary since the API implements synchronization.</span></span>

<span data-ttu-id="01ca7-109">多執行緒需要某種形式的同步處理。</span><span class="sxs-lookup"><span data-stu-id="01ca7-109">Multithreading requires some form of synchronization.</span></span> <span data-ttu-id="01ca7-110">例如，如果在應用程式中執行的多個執行緒必須存取單一裝置內容 ([**>id3d11devicecoNtext**](/windows/desktop/api/D3D11/nn-d3d11-id3d11devicecontext)) ，則該應用程式必須使用某些同步處理機制（例如重要區段）來同步存取該裝置內容。</span><span class="sxs-lookup"><span data-stu-id="01ca7-110">For example, if multiple threads that run in an application must access a single device context ([**ID3D11DeviceContext**](/windows/desktop/api/D3D11/nn-d3d11-id3d11devicecontext)), that application must use some synchronization mechanism, such as critical sections, to synchronize access to that device context.</span></span> <span data-ttu-id="01ca7-111">這是因為處理轉譯命令 (通常是在 GPU) 上執行，並產生轉譯命令 (通常是透過建立物件、資料載入、狀態變更、資料處理) 在 CPU 上進行的處理，通常會使用相同的資源 (紋理、著色器、管線狀態等等) 。</span><span class="sxs-lookup"><span data-stu-id="01ca7-111">This is because processing of the render commands (generally done on the GPU) and generating the render commands (generally done on the CPU through object creation, data loading, state changing, data processing) often use the same resources (textures, shaders, pipeline state, and so on).</span></span> <span data-ttu-id="01ca7-112">跨多個執行緒組織工作需要進行同步處理，以防止某個執行緒修改或讀取其他執行緒正在修改的資料。</span><span class="sxs-lookup"><span data-stu-id="01ca7-112">Organizing the work across multiple threads requires synchronization to prevent one thread from modifying or reading data that is being modified by another thread.</span></span>

<span data-ttu-id="01ca7-113">雖然使用裝置內容 ([**>id3d11devicecoNtext**](/windows/desktop/api/D3D11/nn-d3d11-id3d11devicecontext)) 不是安全線程，但使用 Direct3D 11 裝置 ([**ID3D11Device**](/windows/desktop/api/D3D11/nn-d3d11-id3d11device)) 是安全線程。</span><span class="sxs-lookup"><span data-stu-id="01ca7-113">While the use of a device context ([**ID3D11DeviceContext**](/windows/desktop/api/D3D11/nn-d3d11-id3d11devicecontext)) is not thread-safe, the use of a Direct3D 11 device ([**ID3D11Device**](/windows/desktop/api/D3D11/nn-d3d11-id3d11device)) is thread-safe.</span></span> <span data-ttu-id="01ca7-114">因為每個 **>id3d11devicecoNtext** 都是單一執行緒，所以一次只能有一個執行緒可以呼叫 **>id3d11devicecoNtext** 。</span><span class="sxs-lookup"><span data-stu-id="01ca7-114">Because each **ID3D11DeviceContext** is single threaded, only one thread can call a **ID3D11DeviceContext** at a time.</span></span> <span data-ttu-id="01ca7-115">如果多個執行緒必須存取單一 **>id3d11devicecoNtext**，則必須使用某些同步處理機制（例如重要區段）來同步處理對該 **>id3d11devicecoNtext** 的存取。</span><span class="sxs-lookup"><span data-stu-id="01ca7-115">If multiple threads must access a single **ID3D11DeviceContext**, they must use some synchronization mechanism, such as critical sections, to synchronize access to that **ID3D11DeviceContext**.</span></span> <span data-ttu-id="01ca7-116">但是，不需要使用多個執行緒來使用重要區段或同步處理原始物件來存取單一 **ID3D11Device**。</span><span class="sxs-lookup"><span data-stu-id="01ca7-116">However, multiple threads are not required to use critical sections or synchronization primitives to access a single **ID3D11Device**.</span></span> <span data-ttu-id="01ca7-117">因此，如果應用程式使用 **ID3D11Device** 來建立資源物件，則該應用程式不需要同時使用同步處理來建立多個資源物件。</span><span class="sxs-lookup"><span data-stu-id="01ca7-117">Therefore, if an application uses **ID3D11Device** to create resource objects, that application is not required to use synchronization to create multiple resource objects at the same time.</span></span>

<span data-ttu-id="01ca7-118">多執行緒支援將 API 分成兩個不同的功能區域：</span><span class="sxs-lookup"><span data-stu-id="01ca7-118">Multithreading support divides the API into two distinct functional areas:</span></span>

-   [<span data-ttu-id="01ca7-120">立即和延後轉譯</span><span class="sxs-lookup"><span data-stu-id="01ca7-120">Immediate and Deferred Rendering</span></span>](overviews-direct3d-11-render-multi-thread-render.md)

<span data-ttu-id="01ca7-121">多執行緒效能取決於驅動程式支援。</span><span class="sxs-lookup"><span data-stu-id="01ca7-121">Multithreading performance depends on the driver support.</span></span> <span data-ttu-id="01ca7-122">[如何：檢查驅動程式支援](overviews-direct3d-11-render-multi-thread-support.md) 提供有關查詢驅動程式的詳細資訊，以及結果的意義。</span><span class="sxs-lookup"><span data-stu-id="01ca7-122">[How To: Check for Driver Support](overviews-direct3d-11-render-multi-thread-support.md) provides more information about querying the driver and what the results mean.</span></span>

<span data-ttu-id="01ca7-123">Direct3D 11 已從頭開始設計來支援多執行緒處理。</span><span class="sxs-lookup"><span data-stu-id="01ca7-123">Direct3D 11 has been designed from the ground up to support multithreading.</span></span> <span data-ttu-id="01ca7-124">Direct3D 10 使用 [安全線程層](/windows/desktop/direct3d10/d3d10-graphics-programming-guide-api-features-layers)級，對多執行緒執行的支援有限。</span><span class="sxs-lookup"><span data-stu-id="01ca7-124">Direct3D 10 implements limited support for multithreading using the [thread-safe layer](/windows/desktop/direct3d10/d3d10-graphics-programming-guide-api-features-layers).</span></span> <span data-ttu-id="01ca7-125">此頁面會列出兩個 DirectX 版本之間的行為差異： [Direct3D 版本之間的執行緒差異](overviews-direct3d-11-render-multi-thread-differences.md)。</span><span class="sxs-lookup"><span data-stu-id="01ca7-125">This page lists the behavior differences between the two versions of DirectX: [Threading Differences between Direct3D Versions](overviews-direct3d-11-render-multi-thread-differences.md).</span></span>
