# <a name="creating-tiled-resources"></a><span data-ttu-id="a85fd-103">建立磚資源</span><span class="sxs-lookup"><span data-stu-id="a85fd-103">Creating tiled resources</span></span>

<span data-ttu-id="a85fd-104">建立資源時，藉由指定 [**D3D11 \_ 資源的「 \_ 其他 \_**](/windows/desktop/api/D3D11/ne-d3d11-d3d11_resource_misc_flag) 並排顯示」旗標來建立並排顯示的資源。</span><span class="sxs-lookup"><span data-stu-id="a85fd-104">Tiled resources are created by specifying the [**D3D11\_RESOURCE\_MISC\_TILED**](/windows/desktop/api/D3D11/ne-d3d11-d3d11_resource_misc_flag) flag when you create a resource.</span></span>

<span data-ttu-id="a85fd-105">當您可以使用 [**D3D11 資源的 \_ \_ 其他 \_ 磚**](/windows/desktop/api/D3D11/ne-d3d11-d3d11_resource_misc_flag) 來建立資源時，會在 [磚資源建立參數](tiled-resource-creation-parameters.md)中描述。</span><span class="sxs-lookup"><span data-stu-id="a85fd-105">Restrictions on when you can use [**D3D11\_RESOURCE\_MISC\_TILED**](/windows/desktop/api/D3D11/ne-d3d11-d3d11_resource_misc_flag) to create a resource are described in [Tiled resource creation parameters](tiled-resource-creation-parameters.md).</span></span>

<span data-ttu-id="a85fd-106">建立資源時，系統會在圖形系統中配置非磚資源的儲存體。</span><span class="sxs-lookup"><span data-stu-id="a85fd-106">A non-tiled resource's storage is allocated in the graphics system when the resource is created.</span></span> <span data-ttu-id="a85fd-107">例如，當您呼叫 [**ID3D11Device：： CreateTexture2D**](/windows/desktop/api/D3D11/nf-d3d11-id3d11device-createtexture2d) 來建立2d 材質的陣列時，圖形系統會為這些2d 材質配置儲存區。</span><span class="sxs-lookup"><span data-stu-id="a85fd-107">For example, when you call [**ID3D11Device::CreateTexture2D**](/windows/desktop/api/D3D11/nf-d3d11-id3d11device-createtexture2d) to create an array of 2D textures, the graphics system allocates storage for those 2D textures.</span></span> <span data-ttu-id="a85fd-108">當建立磚的資源時，圖形系統不會配置資源內容的儲存體。</span><span class="sxs-lookup"><span data-stu-id="a85fd-108">When a tiled resource is created, the graphics system doesn't allocate the storage for the resource contents.</span></span> <span data-ttu-id="a85fd-109">相反地，當應用程式建立並排顯示的資源時，圖形系統只會為並排顯示的區域提供位址空間保留，然後再允許應用程式控制磚的對應。</span><span class="sxs-lookup"><span data-stu-id="a85fd-109">Instead, when an application creates a tiled resource, the graphics system makes an address space reservation for the tiled surface's area only, and then allows the mapping of the tiles to be controlled by the application.</span></span> <span data-ttu-id="a85fd-110">磚的「對應」僅為資源中的邏輯磚在記憶體中所指向的實體位置 (若是對應磚則為 **NULL**)。</span><span class="sxs-lookup"><span data-stu-id="a85fd-110">The "mapping" of a tile is simply the physical location in memory that a logical tile in a resource points to (or **NULL** for an unmapped tile).</span></span> <span data-ttu-id="a85fd-111">請勿將此概念與為 CPU 存取對應 Direct3D 資源的概念混淆，其名稱雖相同但卻是完全獨立的概念。</span><span class="sxs-lookup"><span data-stu-id="a85fd-111">Don't confuse this concept with the notion of mapping a Direct3D resource for CPU access, which despite using the same name is completely independent.</span></span> <span data-ttu-id="a85fd-112">您將可視需要個別為每個磚定義並變更對應，並知悉無須一次為一個表面對應所有磚，因此可有效率地使用可用的記憶體量。</span><span class="sxs-lookup"><span data-stu-id="a85fd-112">You will be able to define and change the mapping of each tile individually as needed, knowing that all tiles for a surface don't need to be mapped at a time, thereby making effective use of the amount of memory available.</span></span>

<span data-ttu-id="a85fd-113">本節提供有關如何建立磚資源的詳細資訊。</span><span class="sxs-lookup"><span data-stu-id="a85fd-113">This section provides more info about how to create tiled resources.</span></span>
