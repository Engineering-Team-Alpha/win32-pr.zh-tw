## <a name="resource-types"></a><span data-ttu-id="46b8a-110">資源類型</span><span class="sxs-lookup"><span data-stu-id="46b8a-110">Resource types</span></span>

<span data-ttu-id="46b8a-111">「已認可的資源」的基本概念 (在 direct3d 12 中同時建立初始化的虛擬和實體位址空間) ，但在 Direct3D 12 中可以 teased 虛擬定址 (VA) 和實體定址，以允許應用程式審慎地管理實體記憶體。</span><span class="sxs-lookup"><span data-stu-id="46b8a-111">The basic concept of a "committed resource" (creating both virtual and physical address spaces initialized in managed physical memory) has been around since Direct3D 9, though the virtual addressing (VA) and physical addressing can be teased apart in Direct3D 12 to allow the app to carefully manage physical memory.</span></span>

<span data-ttu-id="46b8a-112">除了認可的資源之外，Direct3D 12 的堆積結構還可啟用其他兩種類型的資源：「放置」和「保留」。</span><span class="sxs-lookup"><span data-stu-id="46b8a-112">In addition to committed resources, the heap construct of Direct3D 12 enables two other types of resource: "placed" and "reserved".</span></span> <span data-ttu-id="46b8a-113">在 Direct3D 11 中，「保留」資源稱為「已並排資源」。</span><span class="sxs-lookup"><span data-stu-id="46b8a-113">In Direct3D 11 a "reserved" resource was known as a "tiled resource".</span></span>

<span data-ttu-id="46b8a-114">保留的資源與放置的資源不同之處在于，保留的資源有自己唯一的 GPU 虛擬位址空間。</span><span class="sxs-lookup"><span data-stu-id="46b8a-114">Reserved resources differ from placed resources in that reserved resources have their own unique GPU virtual address space.</span></span> <span data-ttu-id="46b8a-115">如此一來，您就可以預先進行大量的 VA 空間配置，然後再將 VA 頁面對應到堆積的某些區段，而應用程式會即時重新設定相片順序。</span><span class="sxs-lookup"><span data-stu-id="46b8a-115">This allows a large allocation of VA space up front and then enables mapping of VA pages to certain sections of the heap later, and the application reconfigures the arrangement on the fly.</span></span> <span data-ttu-id="46b8a-116">VA 空間是連續的，而且可以是稀疏的對應至。</span><span class="sxs-lookup"><span data-stu-id="46b8a-116">The VA space is contiguous, and can be sparsely mapped to.</span></span>

<span data-ttu-id="46b8a-117">您可以使用 API 呼叫（例如 [**UpdateTileMappings**](/windows/desktop/api/d3d12/nf-d3d12-id3d12commandqueue-updatetilemappings) ）將保留的資源套用至堆積中的區域，並藉由即時更新頁面資料表，讓應用程式可以使用這些保留資源。</span><span class="sxs-lookup"><span data-stu-id="46b8a-117">The reserved resource can be made to reference regions in the heap with API calls such as [**UpdateTileMappings**](/windows/desktop/api/d3d12/nf-d3d12-id3d12commandqueue-updatetilemappings) and they can be made resident by the app by updating page tables on the fly.</span></span> <span data-ttu-id="46b8a-118">當 VA 範圍對應至 Null 或非常駐堆積時，該部分的資源會視為非常駐。</span><span class="sxs-lookup"><span data-stu-id="46b8a-118">When a VA range is mapped to NULL or a non-resident heap, that portion of the resource is considered non-resident.</span></span> <span data-ttu-id="46b8a-119">當 VA 範圍對應至常駐堆積時，資源的該部分會被視為常駐。</span><span class="sxs-lookup"><span data-stu-id="46b8a-119">When a VA range is mapped to a resident heap, that portion of the resource is considered resident.</span></span> <span data-ttu-id="46b8a-120">堆積會在建立時常駐。</span><span class="sxs-lookup"><span data-stu-id="46b8a-120">Heaps are resident upon creation.</span></span>

<span data-ttu-id="46b8a-121">放置的資源是更簡單的設計，只是堆積特定區域的指標 (例如，5Mb 堆積) 中材質的1Mb 區域。</span><span class="sxs-lookup"><span data-stu-id="46b8a-121">Placed resources are a much simpler design, being simply a pointer to a certain region of a heap (for example, a 1Mb region for a texture in a 5Mb heap).</span></span> <span data-ttu-id="46b8a-122">別名阻礙可讓您使用重迭放置的資源 (指的是 [**CreatePlacedResource**](/windows/desktop/api/d3d12/nf-d3d12-id3d12device-createplacedresource) 和 [**ResourceBarrier**](/windows/desktop/api/d3d12/nf-d3d12-id3d12graphicscommandlist-resourcebarrier)) 。</span><span class="sxs-lookup"><span data-stu-id="46b8a-122">Aliasing barriers enable the use of overlapping placed resources (refer to [**CreatePlacedResource**](/windows/desktop/api/d3d12/nf-d3d12-id3d12device-createplacedresource) and [**ResourceBarrier**](/windows/desktop/api/d3d12/nf-d3d12-id3d12graphicscommandlist-resourcebarrier)).</span></span>

<span data-ttu-id="46b8a-123">保留的資源無法在所有 Direct3D 12 硬體上使用，且放置的資源是合理的回復，不過放置的資源必須是連續的，而且不能是部分常駐的。</span><span class="sxs-lookup"><span data-stu-id="46b8a-123">Reserved resources are not available on all Direct3D 12 hardware, and placed resources are a reasonable fallback, though placed resources must be contiguous and cannot be partially resident.</span></span>

<span data-ttu-id="46b8a-125">在 Direct3D 12 中，當您配置堆積時，您會建立認可資源的實體記憶體層面。</span><span class="sxs-lookup"><span data-stu-id="46b8a-125">In Direct3D 12 when you allocate a heap you are creating the physical memory aspect of a committed resource.</span></span> <span data-ttu-id="46b8a-126">您可以在 Direct3D 12 中選擇更明確的記憶體區段選項， (在影片和系統記憶體) 之間進行選擇。</span><span class="sxs-lookup"><span data-stu-id="46b8a-126">More explicit memory segment choice is available in Direct3D 12 (choosing between video and system memory).</span></span> <span data-ttu-id="46b8a-127">UMA 介面卡只有一個記憶體區段，也就是系統記憶體。</span><span class="sxs-lookup"><span data-stu-id="46b8a-127">UMA adapters only have a single memory segment, system memory.</span></span>

<span data-ttu-id="46b8a-137">由於 Direct3D11 受控記憶體的應用程式，未使用的資源基本上會分頁出。</span><span class="sxs-lookup"><span data-stu-id="46b8a-137">Since Direct3D11 managed memory for applications, unused resources would essentially be paged out.</span></span>

<span data-ttu-id="46b8a-138">選擇最適當的資源維度。</span><span class="sxs-lookup"><span data-stu-id="46b8a-138">Choose the most appropriate resource dimensions.</span></span> <span data-ttu-id="46b8a-139">請考慮資源大小是否適用于應用程式實際執行的情況。</span><span class="sxs-lookup"><span data-stu-id="46b8a-139">Consider whether the size of a resource is appropriate for the situation the application is actually running in.</span></span> <span data-ttu-id="46b8a-140">有些使用者可能會在視窗中執行應用程式，或以800x600 的螢幕解析度執行。</span><span class="sxs-lookup"><span data-stu-id="46b8a-140">Some users may run the application in a window or with a screen resolution of 800x600.</span></span>
